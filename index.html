Configcopier

Designed for .NET developers and development teams, ConfigCopier copies all of the configuration files in and descending from the containing directory (or a specified directory) and places them in a new directory in their original folder structure.

ConfigCopier v1.0

by Zachary Kniebel

Description

ConfigCopier is a simple console application for recursively copying a .NET website's configuration files. It maintains the folder structure in which the config files are stored, so the backups can be simply merged into a new instance of the same project.

Features

  • Ability to specify output directory, source directory, or neither
  • Duplicates folder structure in which config files are currently stored
  • Will also copy disabled config files (e.g. "xxxxx.config.example" or "xxxxx.config.disabled")

Execution

Default Settings:

  • The location of executable will be used as source directory to search for and copy config files from

  • The output directory will be generated inside the source directory, with the name:

    "[source directory name]_configs"

    Note: The output directory will be uniquely named, i.e. no directories will be overwritten

Double-Click Execution:

Running the executable via double-click, i.e. not from the command prompt, will run ConfigCopier with the default settings.

Command Line Execution:

The following are the valid syntaxes for calling ConfigCopier from the command prompt:

ConfigCopier

ConfigCopier.exe

ConfigCopier [output directory path]

ConfigCopier.exe [output directory path]

ConfigCopier [output directory path] [source directory path]

ConfigCopier.exe [output directory path] [source directory path]

Note: If an output path is specified and a directory at that location already exists, it will be overwritten. This only occurs if an output path was EXPLICITELY SPECIFIED
